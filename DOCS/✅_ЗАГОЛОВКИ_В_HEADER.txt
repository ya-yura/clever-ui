📂 СТРУКТУРА CONTEXT:
════════════════════════════════════════════════════════════════════

```typescript
interface DocumentHeaderInfo {
  documentId: string;      // "RCV-001"
  completed: number;        // 5
  total: number;           // 8
}

interface DocumentListInfo {
  title: string;           // "Подбор заказа"
  count: number;           // 8
}

interface DocumentHeaderContextType {
  documentInfo: DocumentHeaderInfo | null;
  setDocumentInfo: (info: DocumentHeaderInfo | null) => void;
  listInfo: DocumentListInfo | null;
  setListInfo: (info: DocumentListInfo | null) => void;
}
```


ИСПОЛЬЗОВАНИЕ В СТРАНИЦЕ СО СПИСКОМ:
```typescript
const { setListInfo } = useDocumentHeader();

useEffect(() => {
  if (!id) {
    // Показываем список документов
    setListInfo({
      title: 'Подбор заказа',
      count: documents.length,
    });
  } else {
    // Показываем конкретный документ
    setListInfo(null);
  }
  
  return () => {
    setListInfo(null);
  };
}, [id, documents.length, setListInfo]);
```


ИСПОЛЬЗОВАНИЕ В HEADER:
```typescript
const { documentInfo, listInfo } = useDocumentHeader();

// В JSX:
<h1 className="text-xl font-medium tracking-wide">
  {listInfo ? listInfo.title : pageInfo.title}
</h1>

{documentInfo ? (
  // Показываем прогресс-бар документа
  <div className="flex items-center gap-2 mt-1">
    <div className="flex-1 max-w-[200px] bg-gray-700 rounded-full h-1">
      <div className="bg-[#86e0cb] h-1 rounded-full" 
           style={{ width: `${progress}%` }} />
    </div>
    <span className="text-xs">
      {documentInfo.completed}/{documentInfo.total}
    </span>
  </div>
) : listInfo ? (
  // Показываем счётчик документов
  <p className="text-xs text-[#a7a7a7] mt-1">
    Всего: {listInfo.count}
  </p>
) : (
  // Показываем subtitle по умолчанию
  pageInfo.subtitle && (
    <p className="text-xs text-[#a7a7a7] mt-1">
      {pageInfo.subtitle}
    </p>
  )
)}
```

🎓 UX ОБОСНОВАНИЕ:
════════════════════════════════════════════════════════════════════

1. DON'T REPEAT YOURSELF
   • Заголовок показывается один раз
   • В самом видном месте (Header)
   • Всегда доступен (не скроллится)

2. MAXIMIZE CONTENT AREA
   • Служебные элементы минимизированы
   • Максимум места для контента
   • Mobile-First подход для ТСД

3. INFORMATION HIERARCHY
   • Самое важное → в Header
   • Контент → основная площадь
   • Ничего лишнего

4. CONTEXT-AWARE HEADER
   • Показывает актуальную информацию
   • Адаптируется к текущему экрану
   • Умный и гибкий

5. UNIFIED APPROACH
   • Единый стиль для всех модулей
   • Единый Context
   • Легко поддерживать
